# AGENTS.md

1. 作業中のユーザーへの報告は必ず日本語にしてください。
2. プルリクエストのコメントを日本語で作成してください
3. プログラムには細かく日本語のコメントを入れてください。モジュールのヘッダーコメントと関数のヘッダーコメントを日本語で作成してください
4. 外部のサービスに依存したコードは最新のドキュメントを確認してから変更を行う。

## 技術メモ作成時のルール
1. 用途は常に「業務アプリケーション」を想定すること。
2. 技術ごとにMarkdownファイルを作成し、`docs/tech_memos/` ディレクトリに保存すること。
3. 以下の項目を明確に区別して記載すること：
   - ユーザーの検証メモまとめ（前提条件、要件、構成など）
   - AIによる補完・補足情報
   - 技術的評価
   - AIコーディングエージェントとの親和性
4. 細かなパターンや代替案などは、ドキュメントの最後に「補足情報」としてまとめて記載すること。
5. サーバーとしての常駐運用だけでなく、クライアントで使うアプリとしての実装・運用についても検討・記載すること。
6. このルールに従い、今後新しい技術のメモを作成する際は、今回確認したような基本事項を再度質問しないこと。
